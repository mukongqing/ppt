## 正文

用AI生成课程汇报PPT的痛，做过pre的都懂：

❌ 配色永远蓝橙渐变，AI味冲出屏幕
❌ 标题永远是"XX方法介绍"，空洞如教科书目录
❌ 引用要么没有，要么糊在页脚像蚂蚁
❌ 每页该讲多久？没有备注，全靠临场发挥

我写了一个 Claude Code Skill，把"论文→PPT"拆成四步管道：

📸 agent1 — 从论文PDF自动提取全部配图
📝 agent2 — 提取结论句+关键数据，逐条标出处
🧠 agent3 — 用金字塔原理生成逐页叙事结构
🎨 agent4 — 锁定 Ocean Gradient 配色，生成pptx

核心思路就一条：**不给AI自由发挥的空间。**

配色锁死三色、标题强制断言句、引用塞进备注不占页面、每页45-60秒演讲备注自动生成。每一步的产出都是结构化文件，哪一步翻车重跑那步就行，不用从头来。

📂 开源：GitHub 搜 `presentation-ppt-maker`
🆓 MIT 协议，随便用
🔧 需要 Claude Code + 三个内置 skill（pdf / ppt-creator / pptx）

## 评论互动预设

Q: "和直接让ChatGPT写PPT有什么区别？"
A: ChatGPT是一次性输出，翻车只能重来。这个管道每步有中间文件，问题定位到具体阶段。而且配色/标题/引用格式全锁死，不会出现AI味的随机发挥。

Q: "需要编程基础吗？"
A: 安装需要命令行操作（git clone + 软链接），但用起来只需要跟Claude说话。如果你的Claude Code已经配好了，零代码。

Q: "免费吗？"
A: 完全开源 MIT，但Claude Code本身是付费产品。
